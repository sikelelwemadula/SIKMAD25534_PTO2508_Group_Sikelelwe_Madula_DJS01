# DJS01 — Podcast Explorer

We were required to build single-page podcast application built with vanilla HTML, CSS, and JavaScript. In this project we needed to demonstrate the use of advanced architectural patterns, focusing on Object-Oriented Programming (OOP) and the SOLID design principles to ensure a scalable and maintainable codebase.

# Key Features
◉ Modular Podcast Architecture: Built using independent classes to manage data fetching, UI rendering, and user interactions.
◉ Responsive UI: A mobile-first design that adapts seamlessly from compact phone displays to expansive desktop views.
◉ Static One-Page Design: Fast, reactive navigation without full page reloads.

# Architectural Principles (SOLID)
The core of DJS01 is built on the five pillars of robust software design:
◉ Single Responsibility Principle (SRP): Each class (e.g., PodcastService, Seasons, podcasts) has one specific role, making the code easier to debug and test.
◉ Open/Closed Principle (OCP): The system is designed to allow new features (like new filtering methods) to be added by extending existing classes rather than modifying the core logic.
◉ Liskov Substitution Principle (LSP): Standardised interfaces ensure that different UI components can be swapped without breaking the application.
◉ Interface Segregation Principle (ISP): Components only depend on the specific data and methods they need, preventing "fat" interfaces and bloated code.
◉ Dependency Inversion Principle (DIP): High-level modules (like the Main App) depend on abstractions rather than low-level details (like specific API endpoints), making the app highly flexible.

# Technical Stack
◉ HTML5: 
Semantic structure with tags for dynamic content.
◉ CSS3:
 Custom properties for theme management and Flexbox/Grid for layout.
◉ Vanilla JavaScript: 
ES6+ classes and modules for a framework-free OOP approach.

# How to Use
1. Clone the repository and open index.html in any modern web browser.
2. Browse Podcasts: Use the main gallery to explore available shows.
